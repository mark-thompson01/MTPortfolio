Basic Bash Shell Scripting

Automate tasks and build reusable tools with Bash: shebang, variables, conditionals, loops, functions, and example scripts.

What is Bash Shell Scripting?

Bash (Bourne Again Shell) is both a shell and command language. Bash scripting means writing sequences of commands in files to automate tasks, from file management to user prompts, loops, and system administration.

Think of it as turning repetitive terminal tasks into small programs that save time, reduce human error, and scale easily.

Why and When is Bash Scripting Beneficial?

  • Automating repetitive administrative tasks (backups, updates).
  • Creating quick solutions for system monitoring or file manipulation.
  • Running sequences of commands without human interaction.
  • Configuring environments or software setups.
  • Saving time and reducing mistakes through automation.

Bash Scripting Fundamentals

#!/bin/bash
# shebang line → tells the system to run this with bash

echo "Hello"
read name
if [ "$name" == "Mark" ]; then
  echo "Welcome back"
else
  echo "Hello $name"
fi
  • #!/bin/bash — shebang, specifies interpreter.
  • echo — print text.
  • read — capture user input.
  • if/else — conditional logic.
  • for/while — loops.
  • functions — reusable blocks of code.
