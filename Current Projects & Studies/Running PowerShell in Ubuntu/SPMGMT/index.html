Running PowerShell on Ubuntu · System & Process Management

System & Process Management

PowerShell provides powerful tools for monitoring system performance, inspecting running processes, sorting output, and filtering results — all from within the Ubuntu terminal. This section demonstrates how to retrieve system information, sort CPU usage, and search for specific processes.

Step 1 — Checking PowerShell Version

Before exploring system commands, it’s useful to confirm which version of PowerShell is installed. Running $PSVersionTable displays your current version details, platform, and environment information.

Step 2 — List Running Processes (Get-Process)

The Get-Process cmdlet retrieves all running processes on the system. This gives a Windows-style process list, even while running on Linux.

Step 3 — Sort Processes by CPU Usage

By piping Get-Process into Sort-Object, you can sort active processes by CPU time or memory usage. This example sorts by CPU descending to reveal the most resource-intensive processes.

Step 4 — Viewing Only the Top CPU Consumers

You can combine Sort-Object with Select-Object -First N to display only the top processes. This is an excellent lightweight alternative to running top or htop.

Step 5 — Filtering for a Specific Process

PowerShell supports pipeline filtering and searching. This example searches for processes related to Firefox using Where-Object.

Step 6 — Using ps aux in PowerShell

PowerShell can run native Linux commands directly. Running ps aux inside PowerShell returns the standard UNIX-style process list.

Step 7 — Searching Native Processes Using grep

Since native Linux commands work in PowerShell, piping into grep lets you quickly search for processes such as cron.

Step 8 — PowerShell + Linux Commands Together

PowerShell can mix native Linux utilities with PowerShell cmdlets, enabling extremely flexible system monitoring workflows.

Step 9 — Sorting and Selecting Process Information

This example highlights sorting processes by different metrics and extracting fields for minimal output — useful for scripting or reporting.

Step 10 — Viewing Process IDs and Session Information

PowerShell exposes additional process metadata such as process IDs (PID), session IDs, CPU time, and memory usage, similar to Windows Task Manager.

Step 11 — Inspecting System Resource Usage

A deeper breakdown of CPU, memory, handles, and thread counts, offering insight into application performance on Ubuntu.

Step 12 — Identifying Background System Processes

Background processes such as system daemons are still visible inside PowerShell, providing system administrators the same visibility they expect on Windows.

Step 13 — Locating Specific Daemons

You can quickly identify service behavior or troubleshoot failures by searching for Linux daemons (like systemd tasks) inside PowerShell.

Step 14 — Viewing Logs Related to System Processes

PowerShell integrates well with log inspection. This example shows logs associated with system-level operations or module builds.

Step 15 — Viewing Log Content with Get-Content

Similar to cat, Get-Content retrieves file contents but supports additional features such as streaming and line-by-line processing.

Step 16 — End-to-End System Insight

The final step demonstrates combining process inspection, filtering, and log viewing to form a full system diagnostic workflow entirely within PowerShell.
