PowerShell Scripting via Nano

In this section, you will walk through writing a complete cross-platform PowerShell system-monitoring script using the nano text editor in Ubuntu. This script detects the operating system, gathers system metrics, and exports structured monitoring data to JSON and CSV formats.

Step 1 — Opening Nano and Creating the Script File

Begin by launching nano to create a new script file named monitor.ps1. Nano opens a blank editor where you can begin writing your PowerShell code.

Step 1

Step 2 — Script Metadata & OS Detection

The script begins by defining metadata such as author and version, followed by logic that determines whether the script is running on Windows or Linux. This allows PowerShell to handle each operating system appropriately.

Step 2

Step 3 — Windows Branch: CPU, Memory, Disk & Network

When Windows is detected, the script gathers system metrics using CIM queries. This includes CPU load, total and free memory, disk statistics, and IPv4 network information.

Step 3

Step 4 — Completing Windows Branch: Network & Uptime

Additional Windows metrics include uptime calculations using Win32_OperatingSystem. These values help build a complete system report.

Step 4

Step 5 — Linux Branch: CPU, Memory & Disk Metrics

In Linux environments, CPU usage is obtained through mpstat or /proc/stat, memory via free -m, and disk usage through df -h.

Step 5

Step 6 — Linux Branch: Network & Uptime Collection

Linux network information is collected using hostname -I, and uptime via uptime -p. This brings the Linux monitoring section to completion.

Step 6

Step 7 — Final Report Creation & Export

Both Windows and Linux branches feed into a unified output object. The script generates an output directory and exports system information to JSON and CSV. A readable summary is printed to the terminal.
