Lab Summary: Active Directory and Group Policy on Windows Server 2016

In this lab, we walked through the process of setting up a Windows Server 2016 virtual machine using Oracle VirtualBox. We then installed the Active Directory Domain Services (AD DS) server role and promoted the server to a domain controller, forming the backbone of a new Windows domain.

We configured the necessary DNS settings within the Windows Server environment to support domain functionality, ensuring proper name resolution for domain-joined devices.

The lab also covered:

  • Creating a new user account in Active Directory Users and Computers
  • Joining a Windows client computer to the domain, which automatically generates a corresponding computer account in Active Directory.
  • Creating and applying Group Policy Objects (GPOs) to both user and computer accounts to enforce policies across the domain.

These tasks form the essential building blocks of managing a Windows domain environment, providing centralized control over users, computers, and security policies through Active Directory.
