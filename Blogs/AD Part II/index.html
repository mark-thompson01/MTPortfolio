Active Directory – Part II: Where AD Starts

From first exposure to installing Active Directory Domain Services in a lab environment.

Oftentimes when Active Directory is introduced and talked about, many people interpret the technology in a more mysterious way. I remember learning about it for the first time when I began my career on the help desk. Early on, I saw it as this unique directory service, without realizing that it’s installed as a server role on Windows Server operating systems, heavily reliant on DNS, and the historical backbone of domain networks.

I walked into my first help desk role on day one and learned that Active Directory facilitates Windows logins as I began performing password resets.

Because of that, I understood Active Directory very differently in the beginning. It wasn’t until I learned how to set it up for the first time using a Windows Server Evaluation version in Oracle VirtualBox — and started learning more about its structure — that I began to understand the technology for what it actually is. That experience helped me realize how many people interpret Active Directory when they’re first introduced to it.

With that being said, I wanted to put together a hands-on demonstration showing where Active Directory actually starts — by installing the AD DS server role on a Windows Server operating system.

This demonstration is being performed using a Windows Server Evaluation version for educational purposes.

If you’re someone new to IT, just getting started, aspiring, or simply interested in learning how to set up Active Directory from scratch in a lab environment, this is a great place to begin.
