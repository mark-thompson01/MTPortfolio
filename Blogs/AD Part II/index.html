Active Directory – Blog Part II: Where AD Starts

From first exposure to installing Active Directory Domain Services in a lab environment.

Installing Active Directory Domain Services

Oftentimes, when AD is introduced and talked about, many can interpret the technology in a more mysterious way. I remember learning about it for the first time back when I began my career on the help desk. Early on, I just saw it as this unique directory service, not realizing that it’s installed as a server role on Windows Server operating systems, heavily reliant on DNS, and the historic backbone of domain networks.

I walked into my first help desk role on day one and learned that AD facilities Windows logins, as I began doing password resets.

So, I understood it very differently in the beginning. It wasn’t until I learned how to set it up for the first time using a Windows Server Evaluation version on Oracle VirtualBox and learned more about its structure, I came to understand the technology more for what it is, which made me realize how many interpret Active Directory when they’re introduced to it for the first time.

With that being said, I thought I would put together a hands-on demonstration on where Active Directory starts. By installing the AD DS server role on a Windows Server operating system and promoting the machine to a domain controller.

This demonstration is being performed on a Windows Server Evaluation version for educational purposes.

If you’re someone new to IT, getting started, aspiring, etc. Or interested in learning how to setup Active Directory from scratch in a lab environment, this is a great starting point.

In this hands-on example, I have an Evaluation version of Windows Server 2022 installed and running on Hyper-V using a Lenovo Thinkpad.

These are the steps you would take to install and setup Active Directory post Windows Server installation.



Step-by-Step Lab Walkthrough

1. Starting out on the Windows Server Manager Dashboard, click on Add roles and features in the quick start listing.


2. The Add Roles and Features Wizard will come up. Click Next.


3. To configure a single server by adding roles, role services, and features, select Role-based or feature-based installation. Then click Next.


4. Select the server the AD DS server role is being installed on, then click Next.


5. In the listing of server roles, select the checkbox for Active Directory Domain Services. click Next.


6. Click the Add Features button for the additional features needed for AD DS.


7. Click Next to the Confirmation slide.

8. Click Yes to allow automatic restarts.


9. Click Install.


10. AD DS Server Role installation in progress.


11. After the installation completes click Promote this server to a domain controller.


12. Select Add a new forest and click Next.


13. Domain Controller Options:

Verify capabilities and set a Directory Services Restore Mode (DSRM) password.


14. Verify the NetBIOS name resolution to the domain.


15. Paths: Paths for AD DS database, logs, and SYSVOL (defaults are fine for lab setups).


16. Review Options


17. Prerequisites Check: Run the check and then click Install.


18. Install & Reboot: The server installs AD DS and restarts automatically.


19. Waiting for the gpsvc service to start.


20. Domain Logon Screen


21. Under the Tools menu on the Server Manager Dashboard is Active Directory Users and Computers.


22. Active Directory Console


23. Expanding the domain hierarchy.


24. Builtin Security Groups - Domain Local


25. Domain Controller(s)

When you install the AD DS server role onto a server, promoting that server into a domain controller, converts that server from a regular server into a domain controller.

A Domain Controller is the server that runs the AD DS Windows Server role. Essentially runs Active Directory.

A Domain Controller can be thought of as the server that provides the horsepower to run the "root" of Active Directory, which is the Forest Root Domain, the logical "root" of the structure.

A single domain can have one more Domain Controllers (DCs), one or more of these computers that run Active Directory.


26. Users OU (Organizational Unit)

The Users OU is where a lot of daily administrative tasks are performed in Active Directory. This is where you create and provision new user account objects, reset passwords, assign applicable group membership to user account objects, and more.


27. Right-click Users OU select New > User to create a new User Account Object.


28. New User Account Object creation.


29. Specify first and lastname attribute details, specify domain logon name. click Next.

30. Specify and create a password for the user account logon.


31. click Finish


32. New User Account Object in Users OU.


An Active Directory implementation begins with installing the AD DS server role on a Windows server operating system and promoting the machine to a domain controller. Here, we have the root of several lab opportunities. See what you can you do.
