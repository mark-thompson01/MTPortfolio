Essential Network Commands

The core commands I use across Windows, Linux, and macOS for connectivity checks, DNS troubleshooting, routing visibility, and more.

Topic: Networking Platforms: Windows · Linux · macOS Focus: Troubleshooting

Overview

The following outlines some Essential Network Commands used across Windows, Linux, as well as macOS.

There are various scenarios where these commands are helpful—from basic connectivity checks to diagnosing DNS problems, addressing IP conflicts, or exploring how traffic flows through your network. This post walks through commonly used commands for IP configuration, name resolution, routing, active connections, and path analysis. It draws from my Cross-Platform Network Commands tutorial in my GitHub Pages portfolio.

The goal is to offer a practical reference—especially for those getting started in IT—by sharing real examples and tools useful in both labs and professional environments.

ping

Ping sends ICMP echo requests to a target and waits for replies, verifying reachability and round-trip time. It’s often the first responder for connectivity checks.

ipconfig / ipconfig /all · ifconfig / ip a

These commands reveal a system’s IP configuration: addresses, subnets, gateways, DNS, and interfaces—core details for verifying network setup and troubleshooting.

arp -a

Lists the ARP cache mapping IP addresses to MAC addresses—useful for spotting duplicate/unauthorized hosts or validating local L2 addressing.

nslookup

Windows DNS lookup tool for querying domain/IP info directly from DNS servers—great for testing resolution and server behavior.

host

Lightweight Linux tool for quick A/AAAA/PTR lookups. Ideal for fast checks without verbose output.

dig

Deep-dive Linux DNS queries with detailed sections (ANSWER, AUTHORITY, ADDITIONAL). Great for validating authoritative responses and timing.

netstat -a / -an

Shows listening ports, active connections, and protocol stats—useful for spotting suspicious listeners or verifying expected services.

tracert (Windows)

Hops and latency on the way to a destination—handy to locate where delays or drops occur.

traceroute (Linux)

Linux equivalent of tracert. Reveals each hop in the path to the target.

pathping (Windows)

Combines ping + tracert, reporting per-hop packet loss and latency—think of it as a path health report.

tracert, traceroute, and pathping are great when you suspect slowdowns or unreachable services. They help pinpoint the hop causing issues.

route print · route -n · ip route

Displays routing tables—the rules your system uses to send packets. Essential when multiple interfaces/default gateways exist or routes seem incorrect.

Takeaways

Mastering these commands equips you to troubleshoot and understand what’s happening on a system or network—across Windows, Linux, and macOS. From verifying connectivity with ping to mapping routes with tracert/traceroute, to inspecting DNS and active connections, these are foundational for IT, security, and system/network administration.

Revisit them in different scenarios, explore options, and fold them into your workflow. The command line doesn’t just tell you what’s wrong—it teaches how systems behave under the hood.
